# Module 1 QA & Deployment Checklist

## 📋 테스트 체크리스트

### 1. 기능 테스트 (Functional Testing)

#### Level 1: 도전, 김밥왕!
- [ ] 게임 로드: 모든 세션에서 정상 로드되나?
- [ ] 닉네임 입력: 특수문자, 공백 처리 정상인가?
- [ ] 라운드 1-4 진행: 각 라운드가 정상 동작하나?
- [ ] CR 계산: 수식이 올바르게 계산되나?
- [ ] 손실함수: 폐기 손실과 품절 손실이 올바르게 계산되나?
- [ ] 리더보드: 실시간 순위 업데이트가 되나?
- [ ] 최종 결과: 4라운드 후 순위와 등급 표시가 정상인가?
- [ ] 재시작: 게임 리셋 후 재시작이 정상인가?

#### Level 2: 편의점 가격 전략
- [ ] 게임 로드: 정상 로드되나?
- [ ] 3개 상품 표시: 올바른 정보 표시되나?
- [ ] 수요 탄성치: 가격 변화에 따른 수요 예측이 정상인가?
  - 김밥 (탄성치 -0.5): 가격 ±10% → 수요 ±5%?
  - 주먹밥 (탄성치 -1.0): 가격 ±10% → 수요 ±10%?
  - 샌드위치 (탄성치 -1.8): 가격 ±10% → 수요 ±18%?
- [ ] 예산 제약: ₩100만 초과 불가인가?
- [ ] 주간 계산: 순이익 계산이 정확한가?
- [ ] 리더보드: 실시간 업데이트되나?
- [ ] 4주 종합: 최종 순위 표시가 정상인가?

#### Level 3: 카페 마크다운 전략
- [ ] 게임 로드: 정상 로드되나?
- [ ] 아침 발주: 3개 상품 발주가 정상인가?
- [ ] 오후 마크다운: 슬라이더 조작이 정상인가?
- [ ] 손익 계산:
  - 아침 매출 = 아침판매 × 아침가격?
  - 오후 매출 = 오후판매 × 할인가?
  - 폐기손실 = 폐기량 × 원가?
- [ ] 20일 시뮬레이션: 매일 진행이 정상인가?
- [ ] 리더보드: 매일 업데이트되나?
- [ ] 주간 토론용 데이터: 제공되나?

### 2. 플랫폼 통합 테스트 (Platform Integration)

#### 로그인 시스템
- [ ] 학생 로그인:
  - [ ] 학번 + 이름 입력 후 로그인 성공?
  - [ ] 필수 입력 필드 검증 작동?
  - [ ] 데이터베이스에 저장되나?
- [ ] 강사 로그인:
  - [ ] 강사ID + 비밀번호 + 학급명 입력 성공?
  - [ ] 잘못된 비밀번호 거부?
  - [ ] 강사 대시보드 표시?

#### 학생 대시보드
- [ ] 모듈 표시:
  - [ ] Level 1-3 카드 표시?
  - [ ] 진행률 표시?
  - [ ] 잠금/진행중/완료 상태 구분?
- [ ] 리더보드:
  - [ ] 실시간 상위 5명 표시?
  - [ ] 본인 순위 표시?
- [ ] 모듈 시작: 각 모듈 링크 작동?

#### 강사 대시보드
- [ ] 학급 통계:
  - [ ] 총 학생 수 표시?
  - [ ] Level별 완료율?
  - [ ] 평균 이익?
- [ ] 학생 진행 현황:
  - [ ] 전체 학생 리스트 표시?
  - [ ] 각 Level 진행 상태 표시?
  - [ ] 개인 최대이익 표시?

### 3. 사용자 경험 테스트 (UX Testing)

#### 접근성
- [ ] 모바일 반응형: 휴대폰에서 정상 표시?
- [ ] 탭 순서: Tab 키로 모든 요소 접근 가능?
- [ ] 폰트 크기: 읽기 편한가?
- [ ] 색상 대비: 시력 약한 사람도 읽을 수 있나?

#### 성능
- [ ] 로딩 속도:
  - [ ] 첫 페이지: 2초 이내?
  - [ ] 게임 시작: 1초 이내?
  - [ ] 라운드 제출: 2초 이내?
- [ ] 메모리: 장시간 플레이해도 슬로우다운 없나?
- [ ] 네트워크: 느린 인터넷(3G)에서도 작동?

### 4. 보안 테스트 (Security)

#### 데이터 보호
- [ ] HTTPS 사용: 모든 연결이 암호화되나?
- [ ] 비밀번호: 강사 비밀번호 평문 저장 안 됨?
- [ ] 세션: 로그아웃 후 뒤로가기로 접근 불가?

#### 입력 검증
- [ ] SQL 주입: 특수문자 입력 안전한가?
- [ ] XSS: 스크립트 입력 차단되나?

## 🚀 배포 단계

### Phase 1: 내부 베타 (1주)
**대상**: 개발팀 + 5-10명 학생

**목표**:
- 기본 게임 플레이 검증
- 심각한 버그 발견 및 수정
- 사용자 피드백 수집

**체크**:
- [ ] 모든 테스트 통과
- [ ] 버그 0개 (Severity: Critical)
- [ ] 사용자 만족도 70% 이상

### Phase 2: 폐쇄 베타 (2-3주)
**대상**: 1개 학급 (40-50명)

**목표**:
- 실제 학급 환경에서 테스트
- 강사 피드백 수집
- 성능 모니터링

**체크**:
- [ ] 게임 완주율 80% 이상
- [ ] 리더보드 실시간 업데이트 정상
- [ ] 강사 기능 만족도 70% 이상
- [ ] 버그 수 < 5개 (모두 Minor)

### Phase 3: 공개 배포 (4주)
**대상**: 전체 학생

**목표**:
- 안정적인 서비스 제공
- 지속적인 모니터링

**체크**:
- [ ] 서버 정상 작동 (99.9% Uptime)
- [ ] 에러율 < 0.1%
- [ ] 사용자 피드백 > 90% 긍정

## 🔄 피드백 루프

### 학생 피드백 수집
**채널**:
- 게임 내 설문 (5개 문항, 2분)
- 주간 강의 후 구두 피드백
- 온라인 설문 (월 1회)

**질문 샘플**:
1. "게임이 재미있었나? (1-5점)"
2. "게임으로 배운 개념이 명확해졌나? (1-5점)"
3. "개선할 점은?"
4. "더 추가되면 좋은 기능은?"
5. "다음 Module은 언제쯤?"

### 강사 피드백 수집
**채널**:
- 주간 강사 미팅
- 월간 설문
- 필요시 1:1 인터뷰

**주요 질문**:
1. "학생 참여도는?"
2. "게임과 강의 연결이 잘됐나?"
3. "채점/평가는 용이한가?"
4. "기술적 문제 있었나?"

### 개선 우선순위
**P1 (즉시)**: 게임 작동 불가 버그
**P2 (1주)**: 심각한 게임 밸런스 문제
**P3 (1개월)**: UI/UX 개선
**P4 (장기)**: 새로운 기능 추가
